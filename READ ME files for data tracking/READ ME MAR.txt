MAR Data Folder - READ ME
The entire dataset in this folder (DATA_MAR) was downloaded from the public Box found at this link:
https://rpi.app.box.com/s/7p8tkqj5ewhtdad2h8kx975i9qg6b7a4/folder/230223517638.

Inside this data folder, there are 3 folders containing samples distributed according to the ratio stated in the report:
a. train folder
b. test folder
c. validation folder

Each of the train, test, and validation folders contains 3 subfolders, one for each source folder from the public Box (we only used the folders body5, body8, and body13). Each of these three subfolders contains 2 directories: gt and corrupted. This means the folders collectively hold sample pairs of {input image, corresponding restored image}.

Additionally, the test folder contains a test_masks directory, which holds a mask for each test sample so that performance metrics can be calculated regionally around the regions of interest (ROI).
According to the split ratio we noted in the report (87% train, 8% test, 5% validation), the distribution in the test folder is as follows: 30 samples (12001-12030) belong to body13, another 80 samples (7001-7080) belong to body8, and another 80 samples (4001-4080) belong to body5.

Note that for the validation set, 19 samples (12031-12049) belong to body13, another 50 samples (7081-7130) belong to body8, and another 50 samples (4081-4130) belong to body5.

Originally, the percentage that each of the three folders constituted out of the total dataset (4748 samples in total) was:
body5 - 42%
body8 - 42%
body13 - 15%

Each of these three was internally divided into train, test, and validation folders according to the percentage ratios mentioned above.
